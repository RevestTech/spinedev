The six-corner moat

Each corner has a leader in isolation. Only Spine lines up all six — the intersection an org actually needs.

LD

Local-deploy

No SaaS runtime. Installs from a tarball, runs as bash daemons + Postgres on your machine, never phones home.

[Only Spine pairs this with full multi-agent SDLC]

MA

Multi-agent

A standing coordinated team of 13 roles — not one smart agent calling subagents on demand. Fan-out up to 130 worker slots.

[Only Spine bounds each role's authority]

RB

Role-bounded

Researchers cannot write code. Engineers cannot deploy. Boundaries enforced by role prompt + auditor cross-check + module-boundary linter.

[Only Spine + MetaGPT enforce this; Spine adds runtime checks]

SG

SDLC-gated

Plan → Build → Verify. Each phase produces a signed artifact: PRD, TRD, Roadmap, BuildArtifact, VerifyFindings. No phase advances without explicit approval.

[Only Spine ships this as editable pipeline-as-data YAML]

RF

Requirements-first

Spine refuses to build until the spec is real. The product role runs a 5-move dialogue protocol with per-project-type intake templates.

[Only Spine has this. Every other tool treats intake as a free-form prompt]

VP

Verify-as-phase

A separate subsystem runs scanners, ephemeral Docker sandboxes, cross-LLM validation (Anthropic + OpenAI), and Platt-scaled confidence calibration.

[Only Spine integrates verification as a first-class SDLC phase]

Who's it for?

Vibecoder

You want to ship software but you can't articulate what you want and you can't debug failures. Spine's product role drags real requirements out of you. The auditor catches the mistakes you can't see.

  • "Build me an app for tracking my team's time off"
  • "I want a personal dashboard for my fantasy football league"
  • "My church needs a sermon-archive site"

Engineering team

A staff engineer running a refactor on a 50-person codebase. The architect queries the Knowledge Graph for blast radius before the TRD. The auditor catches missed callers automatically.

  • "Migrate auth service from session cookies to OAuth2"
  • "Split the monolith's reporting module into a service"
  • "Add OpenTelemetry tracing across 14 microservices"

Org admin / CTO

Roll AI to every employee without losing control. Org bundles inject coding standards, banned patterns, per-user budget caps, and required gates into every install. Audit log is hash-chained.

  • 50-person company, $50/day per user, hard cap
  • Banned: eval(), raw SQL, requests without timeout
  • TRD requires CTO + Compliance both sign for revenue-critical work

Why local-deploy?

Trust

Source code, prompts, and outputs never leave your machine. Compliance arguments collapse — there's nothing to review except the code you audited at install time.

Control

You set the model menu, the budget, the gates, the swarm composition. Spine reads your YAML; the YAML reads no one else's mind.

Cost

You pay your own LLM bills directly. No vendor margin. Cost router can block dispatch when projected spend would exceed your cap.

Customization

Org bundles, custom roles, custom intake templates, editable SDLC pipeline — no forking. Drift detector warns when your bundle lags the org's published version.

Audit

Every LLM call written to an append-only Postgres table: prompt hash, output hash, model, cost, role, user, timestamp, with HMAC hash-chain for tamper detection.

Compared to SaaS alternatives (Devin, Factory, Cursor): excellent at zero-setup turnkey UX, but the runtime, the keys, the data, and the policy live on the vendor's box. If that's the right tradeoff for you, use them — we'll tell you so honestly.
