Critical Security Findings
Critical Hardcoded API Keys in .env and .env.backup .env (lines 11, 16), .env.backup (lines 11, 16) 0.99

Live OpenAI (sk-proj-dve9Xz5...) and Anthropic (sk-ant-api03-atWO7...) API keys are stored in plaintext in .env and .env.backup. While .env is gitignored, .env.backup is not explicitly ignored and could be committed.

Impact: Unlimited unauthorized API usage; potential thousands of dollars in charges.

Fix: 1) Revoke and regenerate both API keys immediately. 2) Add .env.backup and *.env.* patterns to .gitignore. 3) Scan git history for past secret commits. 4) Store production secrets in Azure Key Vault only.
Critical Hardcoded Default Secrets in docker-compose.yaml docker-compose.yaml (lines 102, 169, 16-18) 0.95

Multiple services have hardcoded default credentials: Redis password devRedisPassword123!, JWT secret FCNow-DevOnly-LocalKey-2026-NotForProduction-MinLength32!, and PostgreSQL password fcpassword. While dev-only, these are visible in version control.

Impact: If any dev environment is publicly accessible, all credentials are compromised. Weak defaults could be accidentally deployed to production.

Fix: Use ${VAR:?Error message} syntax to require all secrets via env vars. Validate production secrets at startup in Program.cs (already partially done at lines 682-702).

High CSRF Protection Exemptions for Chat Endpoints backend/Middleware/CsrfProtectionMiddleware.cs (lines 31-34) 0.95

Four chat endpoints (/search/chat, /admin/chat, /advisor/chat, /client/chat) are exempt from CSRF protection with comments like "TESTING ONLY" and "TASK 2.6: Remove before production!"

Critical Performance Findings
Critical N+1 Query Patterns in 40+ Service Methods backend/Services/ClientService.cs (lines 78-131), AdvisorService.cs, PlanService.cs 0.95

GetClientAsync makes 4 separate DB roundtrips (client, relationship, balance, lifecycle) instead of a single query with .Include() chains. Similar pattern across 40+ service methods in AdvisorService, PlanService, etc. Caching masks this but doesn't fix it.

Impact: 4x slower reads when cache is cold. Under load with cache expiry, performance degrades severely.

Fix: Consolidate into single queries: .Include(c => c.Plan).Include(c => c.AdvisorClients).Include(c => c.AccountBalances).Include(c => c.Onboarding). Apply across all services.
Critical Unbounded List Queries — No Pagination in Service Layer backend/Services/AdvisorService.cs (20+ occurrences), ClientService.cs, PlanService.cs 0.95

Multiple service methods use .ToListAsync() without .Skip().Take() or any pagination. With 143 tables and thousands of rows per entity, this causes memory pressure and slow network transfers.

Impact: Loading 10K+ records into memory; AG Grid client-side pagination makes initial page load 3-5 seconds.

Fix: Create PaginationRequest/PaginatedResponse<T> DTOs. Add .Skip().Take() to all list queries. Update frontend hooks to pass pagination state from AG Grid to API.

High Cache Stampede Risk on TTL Expiry backend/Services/ClientService.cs, AdvisorService.cs 0.85

Standard 5-minute cache TTL without probabilistic early refresh. When cache expires, 100+ simultaneous requests all hit the database (thundering herd). RemovePatternAsync("client:*") performs O(N) Redis SCAN.

Fix: Implement probabilistic early expiration (refresh at 80% TTL with p=0.1). Use explicit cache keys instead of pattern matching. Reduce TTL for financial data to 30-60s.
