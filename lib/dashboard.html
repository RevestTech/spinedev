Serving locally

Browsers block fetch() from file:// for security. From your project:

cd .planning/orchestration
python3 -m http.server 8765
# Open http://localhost:8765/dashboard/

Serving only dashboard/ breaks fetch("../agent-handoff/...") (sibling folder is outside the server root). Running from orchestration/ exposes both /dashboard/ and /agent-handoff/. Preset “Installed” = ../agent-handoff/teams (relative to this page under /dashboard/).

If you see JSON like “Route GET:/dashboard/ not found”

You are on a different server (often Fastify / Express). It has API routes only — no static Spine files.

Run Spine’s helper from the repo (serves .planning/orchestration), then open the URL it prints:

make dashboard
# or: bash scripts/serve-dashboard.sh

Use port 60005 by default (SPINE_DASH_PORT=8765 to change). Do not rely on your app’s /dashboard/ path unless you intentionally mount Spine’s HTML there yourself.

Share read-only VPN or LAN access to this static server; the UI only reads Markdown and CSV via HTTP GET. Do not expose write APIs without authentication.
