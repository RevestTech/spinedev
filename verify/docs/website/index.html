Platform Overview

The Problem

AI coding tools generate plausible-looking output nobody can fully trust. Developers spend as much time reviewing AI suggestions as writing code themselves. Agents drift off-task, hallucinate findings, and report false confidence in unverified results.

Tron's Solution

Deterministic tools establish ground truth before the LLM runs. Schema validation catches hallucinations. Multi-agent cross-validation requires 2-of-3 consensus on critical findings. An execution sandbox verifies fixes actually work. Every result carries a calibrated confidence score.

Business Value

Target Market

Enterprise development teams (100-500 developers) with strong QA culture. Security-conscious organizations in fintech, healthcare, and regulated industries.

Revenue Model

Hybrid subscription + usage-based pricing. Land with security auditing, expand to full platform. Per-seat tiers: Starter ($50/dev/mo), Professional ($75), Enterprise ($100).

Competitive Edge

Unlike raw AI output tools, Tron delivers verified results with deterministic confirmation. Zero-drift guarantees, calibrated confidence, and full audit trails for compliance.

Architecture

Tron follows a layered architecture: API gateway → Temporal workflow orchestration → ISO agent framework → verification pipeline → persistence layer. All secrets are loaded from keyvault at runtime; nothing is stored in env files or config.

ISO Agent Framework

Four specialized agents, each scoped to a Blueprint that defines exactly what they can analyze. Cross-validation uses different LLM providers to prevent correlated failures.

SecurityISO

Vulnerability detection. Runs Bandit + Semgrep first, then LLM analysis against OWASP Top 10, CWE database. Provider: Anthropic.

BuilderISO

Code quality and architecture analysis. Checks design patterns, error handling, dependency health. Provider: OpenAI.

PerformanceISO

Performance bottleneck detection. N+1 queries, memory leaks, algorithmic complexity. Provider: Anthropic.

QAISO

Test coverage and quality analysis. Missing edge cases, test isolation, flaky test detection. Provider: OpenAI.

Setup Steps

Configure Environment

Copy the example file and set passwords. In production these come from keyvault; locally you set them directly.

cp .env.example .env

# Edit .env — set at minimum:
POSTGRES_PASSWORD=your-secure-password
REDIS_PASSWORD=your-redis-password
MINIO_USER=minioadmin
MINIO_PASSWORD=minioadmin123
GRAFANA_PASSWORD=admin

Start Infrastructure

Bring up all 18 services. First run pulls images (~5 min). Subsequent starts take ~30 seconds.

docker compose up -d

Verify everything is healthy:

docker compose ps
# All services should show "healthy" or "running"

Run Database Migrations

Apply the schema, agent memory tables, and embedding indexes.

# Apply migrations via the API container
docker compose exec tron-api python -m alembic upgrade head

# Or apply raw SQL migrations directly:
docker compose exec postgres psql -U tron -d tron \
  -f /migrations/001_initial_schema.sql
docker compose exec postgres psql -U tron -d tron \
  -f /migrations/002_agent_memory.sql
docker compose exec postgres psql -U tron -d tron \
  -f /migrations/003_embeddings.sql

Verify API

# Health check
curl http://localhost:13000/health

open http://localhost:13000/api/docs

Usage Guide

Running Your First Audit

Create a project

curl -X POST http://localhost:13000/api/projects \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "my-app",
    "repository_url": "https://github.com/user/repo",
    "languages": ["python", "javascript"]
  }'

Start an audit

This clones the repo, runs deterministic tools, dispatches agents, and executes the full 7-layer pipeline.

curl -X POST http://localhost:13000/api/audits \
  -H "X-API-Key: YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "PROJECT_UUID",
    "check_types": ["security", "quality", "performance"],
    "branch": "main"
  }'

Monitor progress

Connect to the WebSocket for real-time updates, or poll the audit endpoint.

# Poll status
curl http://localhost:13000/api/audits/AUDIT_UUID \
  -H "X-API-Key: YOUR_KEY"

# WebSocket (Socket.IO)
# Connect to ws://localhost:13000/socket.io
# Subscribe to room: "audit:AUDIT_UUID"

Review findings

curl http://localhost:13000/api/audits/AUDIT_UUID/findings \
  -H "X-API-Key: YOUR_KEY"

Each finding includes: severity, confidence score, file/line location, code snippet, verification status, and which deterministic tools confirmed it.

Running Tests

# Full test suite (2,741 tests)
python -m pytest tests/ --ignore=tests/load -q

# Unit tests only
python -m pytest tests/unit/ -q

# Specific module
python -m pytest tests/unit/test_agent_memory.py -v

# Load testing (requires Locust)
pip install locust
locust -f tests/load/locust_config.py --host=http://localhost:13000

Development Workflow

# Install dependencies
pip install -r requirements.txt

# Run API locally (without Docker)
uvicorn tron.api.main:app --reload --port 8000

# Run linting
ruff check tron/ tests/

# Run type checking
mypy tron/ --ignore-missing-imports
